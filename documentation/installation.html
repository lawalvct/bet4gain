Installation Guide

Step-by-step instructions to get Bet4Gain running on your server.

Prerequisites

Before you begin, make sure your server meets these requirements:

Requirement Minimum Version
PHP 8.2+
Composer 2.x
Node.js 18+
npm 9+
MySQL 8.0+ (or MariaDB 10.6+)
Redis 6+ (recommended for production)

Required PHP extensions: OpenSSL, PDO, PDO_MySQL, Mbstring, Tokenizer, XML, Ctype, JSON, BCMath, pcntl, fileinfo, GD or Imagick

Step-by-Step Installation

  1. Upload & Extract Files

    Upload the project archive to your server and extract it to your web root directory (e.g., /var/www/bet4gain). 

    # Example: Extract and set permissions
unzip bet4gain.zip -d /var/www/bet4gain
cd /var/www/bet4gain

  2. Install PHP Dependencies

    Run Composer to install all backend dependencies:

    composer install --optimize-autoloader --no-dev
    Note: Use the --no-dev flag in production to exclude development dependencies like Telescope, Debugbar, and PHPUnit.

  3. Install Node Dependencies & Build Assets 

    npm install
npm run build

    This compiles Vue components, Tailwind CSS, and JavaScript into optimized production assets in public/build/.

  4. Configure Environment

    Copy the example environment file and generate an application key: 

    cp .env.example .env
php artisan key:generate

    Then open .env and configure the essential settings: 

    # Application
APP_NAME=Bet4Gain
APP_URL=https://yourdomain.com

# Database
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=bet4gain
DB_USERNAME=your_db_user
DB_PASSWORD=your_db_password

# Broadcasting (required for real-time game)
BROADCAST_CONNECTION=reverb

# Reverb WebSocket Server
REVERB_APP_ID=your-app-id
REVERB_APP_KEY=your-app-key
REVERB_APP_SECRET=your-app-secret
REVERB_HOST=yourdomain.com
REVERB_PORT=8080
REVERB_SCHEME=https

# Frontend Reverb config (used by Vite)
VITE_REVERB_APP_KEY="${REVERB_APP_KEY}"
VITE_REVERB_HOST="${REVERB_HOST}"
VITE_REVERB_PORT="${REVERB_PORT}"
VITE_REVERB_SCHEME="${REVERB_SCHEME}"

# Payment Gateways (see Configuration page)
PAYSTACK_SECRET_KEY=sk_live_xxxxx
PAYSTACK_PUBLIC_KEY=pk_live_xxxxx

# Cache & Sessions (recommended for production)
CACHE_STORE=redis
SESSION_DRIVER=redis
QUEUE_CONNECTION=redis

    See the Configuration page for full details on all available environment variables.

  5. Create the Database

    Create a MySQL database matching your .env settings, then run migrations and seed initial data: 

    # Create the database
mysql -u root -p -e "CREATE DATABASE bet4gain CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# Run migrations
php artisan migrate

# Seed default data (site settings, demo users, etc.)
php artisan db:seed

  6. Create Storage Symlink

    php artisan storage:link

    This creates a symbolic link from public/storage to storage/app/public for file uploads (avatars, ads, etc.).

  7. Set File Permissions

    chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache

  8. Create Admin User

    Register a normal user account via the app, then promote it to admin: 

    php artisan tinker

# Inside Tinker:
$user = \App\Models\User::where('username', 'your-username')->first();
$user->role = 'admin';
$user->save();

    You can now access the admin panel at /admin.

Web Server Configuration

Nginx (Recommended)

server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/bet4gain/public;
    index index.php;

    add_header X-Frame-Options "DENY";
    add_header X-Content-Type-Options "nosniff";
    add_header X-XSS-Protection "1; mode=block";

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

Apache

The project includes public/.htaccess which handles URL rewriting. Ensure mod_rewrite is enabled: 

sudo a2enmod rewrite
sudo systemctl restart apache2

Required Background Services

Bet4Gain requires three background services to run alongside your web server:

1. WebSocket Server (Laravel Reverb)

Required for real-time game synchronization, live chat, and notifications. 

php artisan reverb:start

2. Game Loop

The game engine that runs continuous crash game rounds:

php artisan game:start

3. Queue Worker (if using Redis/database queue)

php artisan queue:work --queue=default,game
Production Tip: Use Supervisor to keep these processes running and auto-restart them on failure. See the Supervisor configuration examples below.

Supervisor Configuration

# /etc/supervisor/conf.d/bet4gain-reverb.conf
[program:bet4gain-reverb]
process_name=%(program_name)s
command=php /var/www/bet4gain/artisan reverb:start
autostart=true
autorestart=true
user=www-data
redirect_stderr=true
stdout_logfile=/var/www/bet4gain/storage/logs/reverb.log

# /etc/supervisor/conf.d/bet4gain-game.conf
[program:bet4gain-game]
process_name=%(program_name)s
command=php /var/www/bet4gain/artisan game:start
autostart=true
autorestart=true
user=www-data
redirect_stderr=true
stdout_logfile=/var/www/bet4gain/storage/logs/game-loop.log

# /etc/supervisor/conf.d/bet4gain-queue.conf
[program:bet4gain-queue]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/bet4gain/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
user=www-data
numprocs=2
redirect_stderr=true
stdout_logfile=/var/www/bet4gain/storage/logs/queue.log

After creating the Supervisor configs:

sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start all

SSL / HTTPS Setup

For production, HTTPS is required — especially for WebSocket (WSS) connections and payment gateway callbacks. 

# Using Certbot with Nginx
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com

After enabling SSL, update your .env:

APP_URL=https://yourdomain.com
REVERB_SCHEME=https
REVERB_PORT=443

Verifying Installation

  1. Visit https://yourdomain.com — you should see the game page
  2. Register an account and verify you can log in
  3. Check that the game canvas loads and rounds start automatically
  4. Open browser DevTools → Network → WS tab to confirm WebSocket connection
  5. Visit https://yourdomain.com/admin to access the admin panel
  6. Try placing a demo bet to verify the game loop is running
Installation Complete! Head over to the Configuration page to customize game settings, payment gateways, and security options.
