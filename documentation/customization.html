Customization

Tailor Bet4Gain's branding, theme, game visuals, sounds, and architecture to your needs.

Branding

Core branding is managed from Admin Panel → Site Settings — no code changes required:

  • Site Name — Displayed in the header, page titles, and meta tags
  • Site Logo — Upload a logo image (recommended: SVG or transparent PNG, ≤ 200 px height)
  • Favicon — Upload a 32 × 32 favicon
  • Meta Description — SEO description shown in search results

Theme & Colors

The frontend uses Tailwind CSS 4.x with a token-based colour system. Dark/light mode is controlled via a CSS class on the <html> element and persisted to the user's settings.

Key Colour Tokens

Token Usage Where to Edit
primary-500 Buttons, links, active states resources/css/app.css
game-green Win indicators, cashout, profit resources/css/app.css
game-red Loss indicators, crash, danger resources/css/app.css
amber-500 Warnings, pending states resources/css/app.css
surface-light Light-mode card/panel backgrounds resources/css/app.css
surface-dark Dark-mode card/panel backgrounds resources/css/app.css

After editing CSS tokens, rebuild with npm run build.

Game Canvas

The game canvas is rendered by resources/js/Components/Game/GameCanvas.vue. Key visual settings:

Setting Env Variable Default Description
Flying Object GAME_FLYING_OBJECT rocket The emoji/sprite that follows the curve (🚀)
Background GAME_BACKGROUND_THEME default Canvas background style

Canvas Colour Constants

Curve and text colours are defined in resources/js/constants.js: 

CANVAS: {
    CURVE_COLOR: '#22c55e',        // Green curve during running
    CURVE_COLOR_CRASHED: '#ef4444', // Red curve after crash
    TEXT_COLOR: '#ffffff',          // Multiplier text
    GRID_COLOR: 'rgba(255,255,255,0.1)',
}

Edit these values and run npm run build to apply.

Sound Effects

Sound files are located in public/assets/sounds/:

File Trigger
bet-placed.mp3 Player places a bet
cashout.mp3 Player cashes out (success)
crash.mp3 Round crashes
tick.mp3 Multiplier tick (optional)
chat.mp3 New chat message received
win.mp3 Player wins / receives coins

To use custom sounds, replace the files with your own MP3s using the same filenames.

Adding a Payment Gateway

Follow these steps to integrate a new payment provider:

  1. Create a service class in app/Services/ — follow the pattern in PaystackService.php:
    • Implement initializeDeposit(), verifyPayment(), initiateTransfer(), handleWebhook()
  2. Add configuration to config/payment.php under a new key
  3. Add environment variables for API keys, secrets, and URLs
  4. Register webhook and callback routes in routes/api.php
  5. Update the frontend deposit/withdrawal modals to include the new gateway option
  6. Rebuild: npm run build

Extending Chat

Chat is powered by App\Services\ChatService and rendered by resources/js/Components/Chat/ChatBox.vue.

  • New message types: Add entries to the App\Enums\ChatMessageType enum and handle rendering in ChatMessage.vue.
  • Chat commands: Add custom slash-commands in ChatService::processCommand().
  • Media support: Extend the message input component to upload images or GIFs.

...

Stores (Pinia)

resources/js/Stores/ — Centralized state management:

  • gameStore — Game state, round data, multiplier
  • betStore — Active bets, bet placement, cashout
  • userStore — Auth state, user profile
  • walletStore — Balances, transactions
  • chatStore — Messages, online users
  • settingsStore — Site settings, feature flags
  • leaderboardStore — Leaderboard data

Composables

resources/js/Composables/ — Reusable logic:

  • useTheme — Dark/light mode toggle & persistence
  • useSound — Sound effect playback with mute support
  • useToast — Toast notification system
  • usePresence — Online user presence via WebSocket
  • useNotifications — Private channel listener for real-time alerts

Backend Structure

Directory Purpose
app/Services/ Business logic — GameEngine, WalletService, ChatService, PaystackService, NombaService, AntiCheatService, ProvablyFairService
app/Enums/ Type-safe enums — GameRoundStatus, BetStatus, TransactionType, TransactionStatus, ChatMessageType, UserRole, etc.
app/Events/ Broadcast events — GameCrashed, BetPlaced, ChatMessageSent, etc.
app/Models/ 17 Eloquent models with relationships
app/Http/Controllers/ API controllers grouped by feature
app/Filament/ Admin panel resources, pages, and widgets
app/Jobs/ Queued jobs (ProcessWithdrawal, CalculateLeaderboard, etc.)

Build Process

# Development (with HMR hot-reload)
npm run dev

# Production build (minified, optimised)
npm run build

Vite configuration is in vite.config.js. The build outputs to public/build/ with a manifest.json that Laravel uses for asset versioning.

Reminder: Always run npm run build after modifying any frontend file, CSS token, or VITE_* environment variable before deploying.
