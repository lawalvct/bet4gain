Database Schema

Complete database structure — 22 migrations, 17 Eloquent models, and key relationships.

Overview

Bet4Gain uses MySQL 8+ in production (SQLite is supported for local development). The schema is managed through Laravel migrations located in database/migrations/.

Quick Start: Run php artisan migrate to create all tables, or php artisan migrate:fresh --seed for a clean database with default seed data.

Entity Relationships

  • A User has one Wallet and one CoinBalance
  • A User has many Bets, Transactions, and ChatMessages
  • A GameRound has many Bets
  • Each Bet belongs to one User and one GameRound
  • A CoinTransfer belongs to a sender (User) and a recipient (User)
  • A LeaderboardEntry belongs to one User
  • AuditLog, SuspiciousActivity, and LoginLog belong to a User

Running Migrations

# Run pending migrations
php artisan migrate

# Check migration status
php artisan migrate:status

# Fresh install with seed data (drops all tables!)
php artisan migrate:fresh --seed

# Rollback last batch
php artisan migrate:rollback
Warning: migrate:fresh drops all tables and re-runs all migrations. Never run this in production unless you intend to wipe all data.
