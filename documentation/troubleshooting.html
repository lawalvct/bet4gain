Troubleshooting

Common issues and their solutions — organized in a problem → solution format for quick diagnosis.

Tip: Always check storage/logs/laravel.log first — most errors are logged there with full stack traces. You can also use php artisan tinker to debug interactively.

1. Blank Page / White Screen

If you see a completely blank (white) page after opening the app, the issue is usually a missing build step, a configuration error, or a permission problem.

Solutions

  • Check the Laravel log — Open storage/logs/laravel.log and look at the most recent entries for error messages and stack traces.
  • Clear all caches — Run: 
    php artisan config:clear && php artisan cache:clear && php artisan view:clear
  • Build frontend assets — If public/build/ is missing or empty, assets haven't been compiled: 
    npm run build
  • Fix file permissions — The web server needs write access to storage/ and bootstrap/cache/: 
    chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache
  • Generate app key — If APP_KEY is empty in .env: 
    php artisan key:generate

2. WebSocket Connection Failed

If the game doesn't update in real-time, chat doesn't load, or you see WebSocket errors in the browser console, the WebSocket server may not be running or misconfigured.

Solutions

  • Verify Reverb is running: 
    php artisan reverb:start
    You should see output confirming it's listening on a host and port.
  • Check environment variables — Ensure the REVERB_* and VITE_REVERB_* variables in your .env match: 
    REVERB_HOST=127.0.0.1
REVERB_PORT=8080
REVERB_SCHEME=http

VITE_REVERB_HOST=127.0.0.1
VITE_REVERB_PORT=8080
VITE_REVERB_SCHEME=http
  • Rebuild after changing VITE_* variables — Vite inlines these at build time, so you must rebuild: 
    npm run build
  • Check firewall — The WebSocket port (default 8080) must be open in your firewall/security group.
  • SSL/HTTPS setup — If your site uses HTTPS, Reverb must also use a secure connection: 
    REVERB_SCHEME=https
REVERB_PORT=443
VITE_REVERB_SCHEME=https
VITE_REVERB_PORT=443
  • Debug in browser — Open DevTools → Network tab → filter by WS to inspect the WebSocket handshake and any error frames.

3. Game Not Starting / No Rounds

If the game page loads but no rounds are being created or the timer is stuck, the game loop process is likely not running.

Solutions

  • Start the game loop: 
    php artisan game:start
  • Use Supervisor in production — Supervisor will automatically restart the game loop if it crashes. Add a config like: 
    [program:bet4gain-game]
command=php /path/to/artisan game:start
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/path/to/storage/logs/game.log
  • Check game logs — Look in storage/logs/ for game-related error messages.
  • Verify broadcast driver — Ensure your .env has: 
    BROADCAST_CONNECTION=reverb

4. Payment Not Working

If deposits or withdrawals fail, or webhook callbacks are not received, check the payment gateway configuration.

Solutions

  • Verify API keys — Ensure PAYSTACK_SECRET_KEY and PAYSTACK_PUBLIC_KEY are set correctly in .env.
  • Webhook accessibility — Webhook URLs must be reachable from the internet. For local development, use a tunnel service like ngrok.
  • Use test keys for development — Paystack test keys start with sk_test_ and pk_test_.
  • Check logs — Look in storage/logs/laravel.log for payment gateway error responses.
  • Verify APP_URL — Webhook callbacks use APP_URL to construct return URLs. Make sure it's correct: 
    APP_URL=https://yourdomain.com

5. Chat Not Loading

If the chat panel is empty or messages don't appear, it's usually a WebSocket or configuration issue.

Solutions

  • Enable chat — Verify the setting in your .env: 
    GAME_CHAT_ENABLED=true
  • Check WebSocket connection — See Issue #2 above.
  • Check user ban status — Banned users are prevented from joining the chat channel. Check the user's status in the admin panel.
  • Clear browser cache — Hard refresh with Ctrl+Shift+R or clear browser cache and cookies.

6. Admin Panel Access Denied

If you get a 403 Forbidden error when accessing /admin, your user account doesn't have the admin role.

Solutions

  • Set admin role via Tinker: 
    php artisan tinker
> User::find(1)->update(['role' => 'admin'])
  • Run the database seeder — The default seeder creates an admin user: 
    php artisan db:seed
  • Ensure Filament is installed — The admin panel requires Filament: 
    composer require filament/filament

7. Assets Not Loading / 404

If CSS, JavaScript, or images return 404 errors, the assets haven't been compiled or the storage link is missing.

Solutions

  • Compile frontend assets: 
    npm run build
  • Create storage symlink — Required for user uploads and public files: 
    php artisan storage:link
  • Verify build output — Ensure public/build/ exists and contains a manifest.json file.
  • Check web server config — Make sure the document root points to the public/ directory.

8. Migration Errors

Database migration failures are usually caused by incorrect database credentials or conflicting migration state.

Solutions

  • Check migration status: 
    php artisan migrate:status
  • Fresh start (⚠️ WARNING: drops all data): 
    php artisan migrate:fresh --seed
  • Verify database credentials — Check these values in .env: 
    DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=bet4gain
DB_USERNAME=root
DB_PASSWORD=
  • Check MySQL version — MySQL 8.0+ is required for some column types and JSON operations.

9. Leaderboard Not Showing

If the leaderboard page is empty or shows stale data, it may need to be recalculated.

Solutions

  • Force recalculation: 
    php artisan leaderboard:calculate
  • Check cache TTL — The GAME_LEADERBOARD_CACHE_TTL setting controls how long leaderboard data is cached (in seconds). Lower it for more frequent updates.
  • Verify data exists — Leaderboards require completed bets in the database. Check via Tinker: 
    php artisan tinker
> \App\Models\Bet::where('status', 'won')->count()

10. Performance Issues

Slow page loads, laggy game updates, or high server resource usage can often be resolved with caching and configuration tweaks.

Solutions

  • Use Redis in production — Configure Redis for cache, sessions, and queues: 
    CACHE_STORE=redis
SESSION_DRIVER=redis
QUEUE_CONNECTION=redis
  • Cache configuration and routes: 
    php artisan config:cache && php artisan route:cache
  • Scale queue workers — Increase the number of Supervisor queue workers to handle more jobs concurrently.
  • Tune tick rate — The GAME_TICK_RATE setting controls how many WebSocket updates are sent per second during a round. Lower values reduce server and network load but make animations less smooth.
  • Enable OPcache — Ensure PHP OPcache is enabled for significant performance gains in production.

11. Coin Transfer Alert Not Showing

If users don't receive real-time notifications when someone sends them coins, the private channel setup may be misconfigured.

Solutions

  • Verify Reverb is running — The notification relies on WebSockets: 
    php artisan reverb:start
  • Check channel authorization — Ensure routes/channels.php defines the user.{id} private channel.
  • Verify broadcast driver: 
    BROADCAST_CONNECTION=reverb
  • Clear all caches and rebuild: 
    php artisan config:clear && npm run build

Useful Commands

Quick reference table for the most commonly used commands:

Command Purpose
php artisan game:start Start game loop
php artisan reverb:start Start WebSocket server
php artisan queue:work Start queue worker
php artisan leaderboard:calculate Recalculate leaderboards
php artisan config:cache Cache configuration
php artisan route:cache Cache routes
php artisan cache:clear Clear application cache
php artisan migrate Run database migrations
php artisan db:seed Seed default data
php artisan storage:link Create storage symlink
php artisan tinker Interactive PHP shell
