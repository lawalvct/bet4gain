Payments & Wallet

Dual-currency system with real-money deposits, coin exchange, P2P transfers, and withdrawal processing.

Currency System Overview

Bet4Gain operates with a dual-currency model:

Currency Purpose Storage
NGN (Nigerian Naira) Real money — deposits & withdrawals wallets table
Coins Virtual currency — used for betting coin_balances table

Money Flow

Deposit NGN → Buy Coins → Place Bets → Win Coins → Sell Coins → Withdraw NGN

Exchange rates between NGN and Coins are fully configurable via the admin panel or environment variables.

Payment Gateways

Paystack

Primary payment gateway for NGN deposits and withdrawals.

  • Deposits: User initiates a deposit → redirected to Paystack checkout → payment confirmed via webhook → wallet credited.
  • Withdrawals: User enters bank details → bank account resolved via Paystack API → transfer initiated → webhook confirms completion.
  • Test mode: Use sk_test_xxx / pk_test_xxx keys for Paystack sandbox.

Nomba

Alternative payment gateway with similar deposit/withdrawal flow.

Webhook URLs

Gateway Webhook URL Callback URL
Paystack POST /api/payments/paystack/webhook GET /api/payments/paystack/callback
Nomba POST /api/payments/nomba/webhook GET /api/payments/nomba/callback
Important: Webhook URLs must be publicly accessible. Configure them in your payment gateway's dashboard. Make sure APP_URL in your .env file is set to your production domain.

Coin Exchange

Players convert between NGN and Coins through the wallet page:

Action Endpoint Rate Variable
Buy Coins POST /api/wallet/purchase-coins NGN_TO_COIN_RATE
Sell Coins POST /api/wallet/sell-coins COIN_TO_NGN_RATE

Example: If NGN_TO_COIN_RATE = 10, then ₦1,000 buys 10,000 coins. If COIN_TO_NGN_RATE = 10, then 10,000 coins sell for ₦1,000.

P2P Coin Transfer

Players can send coins to other players (gift or transfer) directly from the wallet page.

How It Works

  1. Sender enters the recipient's username → system resolves via POST /api/wallet/resolve-user
  2. Sender specifies amount, type (gift / transfer), and optional note
  3. A configurable fee is deducted (default 2%): COIN_TRANSFER_FEE_PERCENT
  4. Recipient receives coins immediately and gets a real-time notification via WebSocket

Transfer Limits & Guards

Guard Variable Default
Minimum amount COIN_TRANSFER_MIN 100
Maximum per transfer COIN_TRANSFER_MAX 1,000,000
Daily send limit COIN_TRANSFER_DAILY_LIMIT 5,000,000
Minimum account age COIN_TRANSFER_MIN_ACCOUNT_AGE 0 days
Guest access COIN_TRANSFER_GUEST_ENABLED false
Fee percentage COIN_TRANSFER_FEE_PERCENT 2

Transaction Types

Type Description Currency
deposit NGN deposited via payment gateway NGN
withdrawal NGN withdrawn to bank account NGN
purchase_coins NGN converted to coins Both
sell_coins Coins converted to NGN Both
bet Coins wagered on a round Coins
win Coins won from a cashout Coins
refund Bet refund (cancelled or error) Coins
bonus Admin-issued bonus Coins/NGN
coin_transfer_sent Coins sent to another player Coins
coin_transfer_received Coins received from another player Coins

Withdrawal Approval

Withdrawal processing follows a two-tier system:

  • Auto-approve: Withdrawals below the configured threshold (WITHDRAWAL_AUTO_APPROVE_LIMIT) are processed automatically via the payment gateway.
  • Manual approval: Withdrawals above the threshold require admin approval from the admin panel.
Note: All withdrawal requests trigger bank account resolution first via the payment gateway's API to verify the account number and bank code before processing.

Transaction History

Players can view their complete transaction history from the wallet page, with filters for type, status, and date range. A CSV export is available via GET /api/wallet/transactions/export.
