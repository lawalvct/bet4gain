API Reference

Complete endpoint reference for the Bet4Gain REST API. All endpoints are prefixed with /api.

Authentication: Bet4Gain uses Laravel Sanctum session-based authentication. Endpoints marked Auth Required need an authenticated session (cookie). Guest endpoints accept the guest token returned by /api/guest.

Authentication

Method Endpoint Description Auth
POST /api/guest Create a guest (demo) account and receive session Public
POST /api/guest/resume Resume an existing guest session Public

Standard login/register is handled by Laravel Fortify at /login and /register (web routes).

Game — Public

Method Endpoint Description Auth
GET /api/game/state Current game state (phase, multiplier, round data) Public
GET /api/game/history Recent game rounds with crash points Public
GET /api/game/live-bets Active bets in the current round Public
GET /api/game/round/{id} Details for a specific round Public
GET /api/game/round/{id}/bets All bets placed in a specific round Public
GET /api/game/rounds Paginated game round history Public
POST /api/game/verify Verify a round's provably-fair outcome Public

Game — Authenticated

Method Endpoint Description Auth Parameters
POST /api/game/bet Place a bet on the current round Required amount, auto_cashout (optional), slot (1 or 2)
POST /api/game/cashout Cash out an active bet Required slot, bet_id
POST /api/game/cancel-bet Cancel a pending bet (before round starts) Required slot

User

Method Endpoint Description Auth
GET /api/user Current authenticated user with wallet & coin balances Required
POST /api/user/avatar Upload a profile avatar Required
DELETE /api/user/avatar Remove profile avatar Required
PUT /api/user/settings Update user preferences (sound, theme, etc.) Required

Wallet

Method Endpoint Description Auth
GET /api/wallet Get wallet & coin balances Required
GET /api/wallet/transactions Transaction history (filterable by type/status/date) Required
GET /api/wallet/transactions/export Export transactions as CSV Required
GET /api/wallet/gateways List available payment gateways Required
GET /api/wallet/banks List supported banks (for withdrawals) Required
POST /api/wallet/resolve-account Resolve bank account name from account number & bank code Required
POST /api/wallet/deposit Initialize a deposit (returns redirect URL) Required
POST /api/wallet/withdraw Request a withdrawal to a bank account Required
POST /api/wallet/purchase-coins Convert NGN wallet balance to coins Required
POST /api/wallet/sell-coins Convert coins back to NGN wallet balance Required

P2P Transfer

Method Endpoint Description Auth Parameters
POST /api/wallet/resolve-user Resolve a username to verify recipient exists Required username
POST /api/wallet/transfer-coins Send coins to another player Required username, amount, type (gift/transfer), note
GET /api/wallet/transfers P2P transfer history Required —
GET /api/wallet/transfer-config Get transfer limits & configuration Required —

Error Responses

All API errors follow a consistent JSON format:

{
    "message": "Human-readable error description",
    "errors": {
        "field": ["Validation error message"]
    }
}
HTTP Code Meaning
200 Success
401 Unauthenticated — login required
403 Forbidden — insufficient permissions
404 Resource not found
422 Validation error — check errors object
429 Rate limited — too many requests
500 Server error
