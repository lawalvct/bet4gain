Admin Panel

Manage every aspect of your platform from the Filament-powered administration dashboard.

Accessing the Admin Panel

The admin panel is available at /admin. Only users with the admin role can sign in. After running the database seeder a default admin account is created — see Installation for details.

Tip: You can promote any user to admin via Tinker: php artisan tinker → User::find(1)->update(['role' => 'admin'])

Dashboard

The dashboard shows key metrics at a glance via three built-in widgets:

  • StatsOverviewWidget — Cards showing total users, active bets, today's revenue, total rounds played, and pending withdrawals.
  • RevenueChartWidget — Interactive line chart plotting platform revenue over the last 30 days.
  • LatestBetsWidget — Live-updated table of the most recent bets with player, amount, multiplier, and payout.

User Management

The Users resource lets you list, search, filter, and edit every registered account.

Capabilities

  • Search by username, email, or ID
  • Filter by role (user, admin, moderator) and status (active / banned)
  • Ban / un-ban a user instantly
  • Adjust wallet balance & coin balance
  • Change a user's role
  • View full bet history and transaction log per user
  • Soft-delete an account

Game Round Management

Every crash round is recorded and viewable from the Game Rounds resource.

  • Browse all rounds with crash point, status, start/crash timestamps
  • Filter by status: waiting, betting, running, crashed
  • Click into a round to see all bets placed, cashout times, profit/loss per player
  • View provably-fair seeds (server seed, client seed, nonce, hash)

Transaction Management

The Transactions resource provides a complete financial audit trail.

Filters

Filter Options
Type deposit, withdrawal, purchase_coins, sell_coins, bet, win, refund, bonus, coin_transfer_sent, coin_transfer_received
Status pending, completed, failed, reversed
Gateway paystack, nomba
Date Range Custom start / end dates

Withdrawal Approval

Pending withdrawal requests appear with an Approve / Reject action. Withdrawals below the auto-approve threshold are processed automatically; those above require manual admin approval.

Security Management

IP Restrictions

Manage an IP blacklist and whitelist. Define optional expiry dates and reasons.

Suspicious Activity

The anti-cheat system automatically flags anomalies (multi-accounting, rapid withdrawals, bot-like betting patterns). Review and resolve suspicious activities from this resource.

Audit Log

Every admin action (create, update, delete) is logged with old/new values, the acting admin's ID, and IP address.
